Run the Objection Handling exercise from `~/.claude/skills/pitch/SKILL.md` (Stage 4: Objection Handling).

1. Ask the user for: company name, stage, known weaknesses (e.g., no technical co-founder, early traction, high churn)
2. Identify their top 5 most likely objections from the 20 common VC objections
3. Generate customized response scripts using the Acknowledge → Reframe → Evidence → Bridge framework
4. For each response, include a specific data point or proof they should prepare
5. Generate a pre-meeting prep checklist tailored to their situation
6. Flag any objections where their current answer is weak — suggest what data to gather before pitching
7. Practice: present each objection and let the user rehearse their response
